Core Statistical Insight

Sequence amplification: |z_seq| ≈ |z_daily| × √L

When a transaction type consistently deviates from its training baseline by k standard deviations per day, a window of L days produces a sequence z-score magnitude of approximately k√L. A sustained MEDIUM alert (−2σ/day) escalates to HIGH after just 3 consecutive days (√3 × 2 = 3.46σ). This class of slow-burn attack is structurally invisible to single-day z-score analysis alone.

Revised Pipeline Architecture

The sequence path runs in parallel with the existing single-day path and merges at Anomaly Identification. It activates only after L days have accumulated in the rolling buffer; below that threshold it is silently skipped, leaving the existing pipeline unaffected.

SCPI Mathematics

Sequence CPI Definition

For transaction type k, the Sequence Churn Proximity Index over a window of L days ending at day t is the geometric mean of those daily CPIs:

SCPI_k(t, L) = ( ∏ CPI_k(t−i+1) )^(1/L) for i = 1 … L · computed in log space for numerical stability
z_k^seq = ( SCPI_k − μ_k ) / σ_k^seq σ_k^seq = σ_k / √L

The σ/√L adjustment follows from the delta-method standard error of the geometric mean, valid when σ ≪ μ — which holds for any stable QBO training baseline. The same HIGH / MEDIUM / LOW band thresholds (|z| > 3, > 2) apply unchanged. No Training Matrix schema changes are required.

Slow-Burn Vendor Scenario (QBO Case Study)

Baseline: Vendor CPI μ = 0.995, σ = 0.005 (stable company, ~200 vendors, ~1–2 new per month). Attack: 3 new shell vendors created each day for 7 days. Each day individually produces a MEDIUM alert at most; no single day reaches HIGH. The 5-day SCPI crosses HIGH from day 5. The amplification factor √5 ≈ 2.24 turns a sustained −2.6σ peak into a −4.9σ sequence signal.

|z| Trajectory — Vendor Type (φ18)

HIGH threshold at |z| = 3.0. The single-day line never exceeds 2.60 (MEDIUM). The sequence line crosses 4.7 at day 5. Zero values days 1–4 indicate the buffer has not yet accumulated L = 5 days.

Implementation Phases

Rolling Buffer Phase 1

Modify NGameCpiAnalysisAgent to append each day's CPI and rate arrays to the rolling buffer JSON after extraction.

No other agents change. Buffer is write-once per day and read-only from the sequence path. Existing pipeline is unaffected.

Sequence CPI Agent Phase 2

New NGameSequenceCpiAgent with geometric mean computation and σ/√L adjustment. Fully unit-testable with mock buffer data; no API calls required.

Pipeline Integration Phase 3

Extend NGameChurnComparisonAgent, wire run_sequence_analysis.py and the demo script. Optionally surface sequence results in the management dashboard JSON alongside existing single-day results.

The Training Matrix schema is unchanged throughout all phases. The sequence path is fully additive — existing single-day analysis, training flows, and demo scenarios remain intact. Window length L is configurable at runtime with no retraining required.
