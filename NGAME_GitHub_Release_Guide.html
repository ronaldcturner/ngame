Phase 3 — Add the Apache 2.0 License

Apache 2.0 is the standard open-source license that lets adopters use, modify, and distribute NGAME freely while protecting you from liability. A LICENSE file has already been created at the repo root with the copyright line set to Copyright 2026 Ron Turner.

For reference, the commands to download it from scratch are:

# macOS / Linux
curl -o LICENSE https://www.apache.org/licenses/LICENSE-2.0.txt
# Windows PowerShell
Invoke-WebRequest -Uri https://www.apache.org/licenses/LICENSE-2.0.txt -OutFile LICENSE

After downloading, open the file and replace the boilerplate copyright line near the bottom:

   Copyright 2026 Ron Turner
Once the LICENSE file exists, add this header comment to the top of each .py file (optional but good practice):
# SPDX-License-Identifier: Apache-2.0

Phase 4 — Rename the README

GitHub looks for exactly README.md at the repo root to display on the project page. Your original file was named README (Hi-Level description of NGAME).md (spaces and parentheses in the name). It has already been renamed to README.md.

Phase 6 — Initialize Git Locally and Push

Run these commands from inside your project folder (NGAME-POC). Open Terminal (Mac) or Git Bash / PowerShell (Windows).

6-A. Navigate to the project

cd ~/Developer/Projects/NGAME-POC        # macOS
cd C:\Users\<YourName>\...\NGAME-POC   # Windows — adjust to your actual path

6-B. Initialize the local repository

git init
git branch -M main

6-C. Link to GitHub

git remote add origin https://github.com/<your-github-username>/ngame.git

Replace <your-github-username> with your actual GitHub username.

6-D. Stage files and verify

Add all files, then inspect what will be committed before you commit.

git add .

# Review what is staged — scan for anything sensitive before committing
git status
Security check

Read the git status output carefully. You should NOT see quickbooks_config.json, wave_config.json, .env, any .venv/ or venv/ path, or anything from SCRAP/. If you do, fix .gitignore first, then run git reset and re-add.

6-E. First commit

git commit -m "Initial open-source release of NGAME v1.0

Apache 2.0 licensed. Agent-driven fraud analytics pipeline
for QuickBooks and Wave accounting data."

6-F. Push to GitHub

git push -u origin main

Git will prompt for your GitHub credentials on first push. If you have two-factor authentication enabled (you should), create a Personal Access Token (PAT) at:
github.com → Settings → Developer Settings → Personal access tokens → Tokens (classic)
Use the token as your password when prompted.

6-G. Verify on GitHub

Refresh your GitHub repo page. You should see the file tree and the README displayed below it.

Day-to-Day Git Workflow (After Initial Push)

Each time you improve NGAME and want to publish the changes:

git add .
git status                        # verify what changed
git commit -m "Brief description of what you changed"
git push

To pull updates from GitHub to another machine: git pull

Adopter Installation Guide — macOS  macOS 12+

These are the instructions you will include in your README (or a separate INSTALL.md) for security practitioners installing NGAME on a dedicated surveillance Mac.

Step 1 — Prerequisites

RequirementHow to get itNotes
macOS 12 Monterey or laterAlready installed
Python 3.10 or laterpython.org/downloads or brew install python@3.13Check: python3 --version
Gitxcode-select --install (Terminal prompt appears)Check: git --version
QuickBooks Online account or Wave accountYour accounting platformCredentials needed in Step 5
Ollama (optional — LLM analysis)ollama.com/downloadEnables LLM-powered risk narrative

Step 2 — Clone the Repository

# Open Terminal (Cmd+Space → type "Terminal")
git clone https://github.com/<your-github-username>/ngame.git
cd ngame

Step 3 — Create a Python Virtual Environment

python3 -m venv .venv
source .venv/bin/activate

Your Terminal prompt will show (.venv) when the environment is active. You must activate it every time you open a new Terminal session before running NGAME.

Step 4 — Install Python Dependencies

pip install --upgrade pip
pip install -r requirements.txt

This installs ~30 packages (numpy, pandas, rdflib, scikit-learn, Flask, etc.). It takes 2–5 minutes on first install.

Step 5 — Configure Your Accounting Credentials

Wave (recommended for simplest setup):

cp wave_config.example.json wave_config.json

Open wave_config.json in any text editor and fill in your values:

{
  "wave_graphql": {
    "endpoint": "https://gql.waveapps.com/graphql/public",
    "access_token": "YOUR_WAVE_ACCESS_TOKEN",
    "business_id": "YOUR_WAVE_BUSINESS_ID"
  }
}

QuickBooks Online:

cp quickbooks_config.example.json quickbooks_config.json

Open quickbooks_config.json and enter your QuickBooks OAuth credentials (Client ID, Client Secret, Company ID). See NGAME_WAVE_GRAPHQL_README.md and the QuickBooks Developer Portal for credential setup steps.

Security

wave_config.json and quickbooks_config.json are listed in .gitignore. They will NEVER be committed to Git. Store them only on your local surveillance machine.

Step 6 — (Optional) Install and Start Ollama

Ollama provides the local LLM that generates narrative risk assessments. Without it, NGAME still runs but skips the LLM analysis stage.

# After installing from ollama.com:
ollama serve &           # starts the server in background
ollama pull llama3       # or whichever model you prefer

Step 7 — Run NGAME

Training Mode (first 30 days — build baseline):

python3 run_training_flow.py

Fraud Analysis Mode (once baseline is established):

python3 run_fraud_analysis.py

Auto Mode (selects mode based on matrix state):

python3 ngame_dual_mode.py

Web Dashboard (open in browser at http://localhost:5001/dashboard):

cd ngame_ui
python3 app-simple.py
For the dashboard to show data, run at least one of the analysis commands above first to generate the JSON output files that the dashboard reads.

Step 8 — Daily Automation (cron)

To run NGAME automatically each morning at 7 am:

crontab -e
# Add this line (adjust paths to your actual install location):
0 7 * * * /path/to/ngame/.venv/bin/python3 /path/to/ngame/ngame_dual_mode.py >> /path/to/ngame/logs/ngame_daily.log 2>&1

python -m venv .venv

PowerShell execution policy error?

If you see an error about scripts being disabled, run this one-time fix:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Then re-run .venv\Scripts\Activate.ps1

Your prompt will show (.venv) when the environment is active. You must activate it every time you open a new PowerShell window before running NGAME.

Step 4 — Install Python Dependencies

pip install -r requirements.txt

This installs ~30 packages. Allow 3–7 minutes. If any package fails with a compiler error, install Microsoft C++ Build Tools from visualstudio.microsoft.com/visual-cpp-build-tools/ and retry.
