5.1  Evaluation Framework

Hevner et al. [2004] identify five methodologically distinct approaches to evaluating design science artifacts: (1) observational, including case studies and field studies; (2) analytical, including static analysis, architecture analysis, and optimization; (3) experimental, including controlled experimentation and simulation; (4) testing, including functional and unit testing; and (5) descriptive, including informed argument and scenario construction. They further distinguish between naturalistic evaluation — conducted in real-world settings — and artificial evaluation — conducted in controlled laboratory conditions. The DSRM (Peffers et al. [2008]; Geerts [2011]) requires that the evaluation activity compare the artifact's observed performance against the objectives stated in Activity 2. This section applies four of Hevner et al.'s five evaluation categories — testing, analytical, observational, and descriptive — to NGAME, presenting evidence from each in turn. Experimental evaluation, which would require a prospective controlled trial on live fraudulent data, is deferred to future work and is acknowledged as a limitation in §5.8.

The evaluation of NGAME is structured around three independent sources of evidence: (i) a controlled anomaly injection test suite executed against a synthetic transaction stream; (ii) analytical decomposition of the CPI vector comparison metrics produced during live operational runs; and (iii) an observational field deployment against a consenting nonprofit organization's production QuickBooks Online account. Together, these sources address two questions central to the DSRM evaluation activity: Does the artifact operate correctly and reliably (technical validity)? And does the artifact detect what it is designed to detect with measurable accuracy (purpose validity)?

5.2  Metric Definitions for the NGAME Detection Task

Before reporting results, it is necessary to operationalize the standard information retrieval metrics — precision, recall, and false-positive rate — in terms specific to the NGAME detection task. The detection task is defined as follows: given a daily CPI phi array of N dimensions, NGAME ranks each dimension by its absolute deviation from the trained baseline and flags the top-k dimensions as anomalous. In the current instantiation, N = 17–18 (depending on the source system's transaction-type coverage) and k = 3.

A True Positive (TP) is defined as a flagged dimension that corresponds to a transaction type in which a synthetic or genuine anomaly was knowingly introduced. A False Positive (FP) is a flagged dimension that corresponds to a transaction type experiencing only normal variation. A True Negative (TN) is a dimension not flagged that experienced only normal variation. A False Negative (FN) is a dimension not flagged that contained an introduced anomaly. Under this formulation:

MetricFormulaInterpretation for NGAME
PrecisionTP / (TP + FP)Of the top-k CPI dimensions flagged as anomalous, what fraction correspond to dimensions in which an anomaly was actually introduced?
Recall (Top-k)TP / (TP + FN)Of all dimensions in which an anomaly was introduced, what fraction appear in NGAME's top-k flagged set?
False Positive RateFP / (FP + TN)Of all normal-variation dimensions, what fraction are incorrectly included in the top-k alert set?
Cosine Similaritycos(φtoday, φbaseline)Aggregate vector-level similarity between the current-day phi array and the trained baseline; values approaching 1.0 indicate behavioral stability.
Root Mean Squared Error (RMSE)√( Σ(φi,today − φi,avg)² / N )Mean magnitude of element-wise deviation across the full phi array; sensitive to the scale of individual spikes.

Two clarifications govern interpretation throughout this section. First, because NGAME's detection is proportional (phi values are frequency ratios, not dollar amounts), precision and recall measure the system's ability to rank the correct transaction-type dimensions, not the dollar value of any particular fraudulent transaction. Second, the system is designed as a triage instrument: its output is an alert to the FRP, not a conclusive determination of fraud. Consequently, a moderate false-positive rate is preferable to under-detection; the cost asymmetry favors recall over precision in the system's intended use context [cf. ACFE 2022 median detection lag of 12 months under no continuous monitoring].

5.3  Testing Evaluation: The Anomaly Injection Test Suite

Hevner et al. [2004] describe testing evaluation as the use of a well-specified test suite to identify technical faults in the artifact. For NGAME, a dedicated Anomaly Injection Agent (ngame_anomaly_injection_agent.py) was constructed specifically to support this form of evaluation. The agent creates synthetic vendor identities, generates anomalous transaction streams against a controlled QuickBooks Online sandbox environment, and records the injected patterns for ground-truth comparison. The test suite was executed on 25 April 2026 and produced a validated test record (anomaly_injection_test_suite_final_report.json).

The suite exercised five distinct anomaly pattern classes, representing recognizable behavioral signatures of occupational fraud as documented in the ACFE Occupational Fraud and Abuse taxonomy [ACFE 2022]. For each pattern, five synthetic vendor identities were created and 59 anomalous transactions totaling $1,695,096 in synthetic value were generated and injected into the test stream. The test harness then executed the full NGAME pipeline — extraction, ontology construction, CPI analysis, churn comparison, anomaly identification, account mapping, LLM analysis, and management warning — and recorded pass/fail status for each of the test suite's eight validation checks.

All four phases of the test suite completed with status COMPLETED or PASS, and all eight validation checks returned true. The pipeline was confirmed operational across all component agents: anomaly injection, vendor creation, transaction generation, and reporting. These results satisfy the technical validity criterion of Hevner et al.'s [2004] testing evaluation category: the artifact executes correctly and completely on a well-specified test input.

Interpretive note for the author: The injection suite as executed confirms pipeline operability and end-to-end component integration. It does not yet provide element-level precision and recall because the test harness records overall pass/fail status rather than recording which specific CPI dimensions were flagged as highest-ranked anomalies for each injected pattern. A forthcoming revision to the test harness — recording top-k CPI rankings per injected pattern — will enable formal precision/recall computation in a subsequent study. This limitation should be stated explicitly in §5.8.

5.4  Analytical Evaluation: CPI Vector Decomposition on Live Data

Analytical evaluation, as defined by Hevner et al. [2004], involves mathematical decomposition or architectural analysis of the artifact's properties without necessarily conducting a field trial. For NGAME, this form of evaluation is operationalized by examining the full suite of distance and similarity metrics produced by the Churn Comparison Agent on a mature operational run — one in which the training matrix is populated from genuine historical transaction data rather than synthetic inputs.

On 28 April 2026, NGAME was executed in Fraud Analysis Mode against a QuickBooks Online account with a populated training baseline. The system analyzed an 18-element phi array (φ₁–φ₁₉, with one index reserved). The comparison against the trained mean produced the following aggregate statistics:

The concentration of deviation in three specific dimensions — Vendors (φ₁₉: +30.2%), Contractors (φ₁₅: +47.6%), and ChartOfAccounts (φ₁₇: +102.2%) — is analytically significant for two reasons. First, these three transaction types are among the highest-weighted dimensions in the system's a priori misappropriation risk ranking, as determined independently by the composite risk scoring model embedded in ngame_misappropriation_risk_ranking.json. In that model, Vendors earned the highest composite risk score (0.878, weighted: risk 0.35, frequency 0.20, ACFE evidence prevalence 0.45), reflecting its taxonomic alignment with Billing Schemes — the most prevalent occupational fraud category, accounting for 22% of ACFE-reported schemes. The convergence of anomaly detection (a statistically-derived signal from daily phi deviation) and a priori risk ranking (a theory-derived expectation from the ACFE taxonomy) across the same Vendors dimension constitutes convergent validity evidence for the CPI measurement instrument.

Second, the analytical evidence is corroborated by an independent measurement channel: the day-over-day phi coefficient similarity analysis (phi_values_array.json). This analysis applies a Jaccard-family similarity coefficient to the current-day and prior-day ontology snapshots for each transaction type separately. On the same day as the NGAME Fraud Analysis run, the phi coefficient for the Vendors dimension was 0.863 and for Contractors was 0.855 — both rated "Very Strong" but notably below the 1.000 recorded for stable dimensions such as Invoices, Bank Transactions, Bill Payments, Estimates, Recurring Transactions, and Sales Transactions. The fact that the two dimensions with sub-unity phi coefficients precisely overlap with two of the three HIGH-deviation dimensions in the z-score analysis provides measurement convergence across two independent analytical paths within the same artifact run.

5.5  Observational Evaluation: Nonprofit Field Deployment

Observational evaluation, as described by Hevner et al. [2004], involves monitoring the artifact in its intended use environment. For NGAME, the primary observational context was a deployment against the production QuickBooks Online account of a consenting nonprofit organization, with the researcher acting in the role of the Financially Responsible Person (FRP). This arrangement satisfies the naturalistic component of observational evaluation: the artifact operated on live, unmodified financial data produced by an ongoing business operation rather than a synthetic or controlled dataset.

The field deployment proceeded in two phases. During the Training Phase, NGAME was executed daily over a 30-day baseline accumulation window. The system's Training Mode executed the full Phase I pipeline (data extraction, ontology construction, CPI computation) each day and appended the resulting phi array as a new row in the training matrix. The training matrix accumulated a mean and standard deviation for each of the 17–18 phi dimensions over this window, establishing the organization's normal behavioral fingerprint. The successful accumulation of a complete training matrix confirmed that the artifact's data extraction pipeline (OAuth authentication, QBO API query, TTL ontology construction) operates reliably in a production OAuth environment with a real customer account.

During the Fraud Analysis Phase, the system transitioned automatically via the dual-mode entry point (ngame_dual_mode.py) upon the training matrix exceeding the configured minimum row threshold. The fraud analysis pipeline executed without modification and produced the quantitative results reported in §5.4. From an observational standpoint, the key finding is that the artifact ran to completion, delivered output artifacts (management_dashboard.json; NGAME_Fraud_Analysis_readable.json), and produced a management-facing warning report that was reviewed and found interpretable by the FRP without technical explanation. This satisfies Objective O6 of the solution objectives stated in §4: the artifact must produce management-facing warning artifacts that require no accounting or technical expertise to interpret.

A further observational finding concerns the ChartOfAccounts dimension (φ₁₇), which recorded the largest percent deviation (+102.2%) in the April 28 run. Observational review of the organization's actual QBO account during that period revealed that new GL accounts had been added by the bookkeeper during the analysis window — a legitimate administrative action that explains the elevated ChartOfAccounts phi value. This observation is analytically important: it demonstrates both the sensitivity of the artifact (it correctly flagged a structural change to the chart of accounts) and the need for FRP judgment in interpreting alerts (a chart-of-accounts modification is not inherently fraudulent but warrants review, consistent with the artifact's design as a triage instrument). The artifact performed exactly as intended: it surfaced an item requiring managerial attention, and the FRP's review resolved the alert as a benign administrative action. No false negative was incurred.

5.6  Descriptive Evaluation: Comparative Argument and Scenario Analysis

Descriptive evaluation, in Hevner et al.'s [2004] taxonomy, proceeds by informed argument: comparing the artifact's capabilities against those of existing solutions, constructing representative scenarios, and demonstrating that the artifact addresses identified gaps more effectively than prior work. This form of evaluation is particularly appropriate for demonstrating the social and organizational relevance of the artifact — the "utility" criterion of design science research [Hevner et al. 2004, p. 77].

The primary comparators for NGAME are the two established audit analytics platforms most widely used in the accounting profession and taught in university accounting programs: ACL Analytics (Diligent) and IDEA Data Analysis (CaseWare). Both are mature, well-validated tools with published academic evaluations [cf. Bovee et al. 2005, who specifically evaluated agent-based AIS analytics tools]. The comparative analysis demonstrates that NGAME does not simply replicate existing tools in a new platform context. It addresses a structural gap: the existing audit analytics tools presuppose the involvement of a qualified auditor operating in a post-period review mode. NGAME's design resolves this presupposition by replacing the auditor-as-operator with a behaviorally-trained automated pipeline, and by replacing the auditor's interpretive judgment with an LLM-assisted risk assessment aligned to a published fraud taxonomy. The result is an artifact that makes continuous behavioral surveillance accessible to the FRP — a population that has historically had no equivalent tool.

To further ground the descriptive evaluation, consider the following scenario drawn from the April 28, 2026 operational run. On that date, the Vendors dimension registered a z-score of 5.0 — a deviation that, under the assumption of normality, would occur by chance with probability p < 0.0001. Had this been a genuine billing scheme in which a bookkeeper added a fictitious vendor and processed a disbursement, the FRP under the conventional monitoring regime (periodic review, no continuous surveillance) would likely not discover the deviation until the next bank reconciliation or annual audit — a lag consistent with the ACFE's reported median detection period of 12 months [ACFE 2022]. Under NGAME, the FRP would receive a management dashboard alert on the evening of the same business day on which the anomalous vendor activity occurred. This temporal compression — from months to hours — is the artifact's primary practical contribution to the FRP's internal control environment.

5.8  Limitations and Future Evaluation Work

Four limitations bound the evaluation claims presented in this section and must be acknowledged explicitly.

First, the primary experimental limitation is the absence of a prospective controlled trial involving genuine fraudulent transaction data. The injection suite tests pipeline operability under synthetic anomalies, and the field deployment provides observational evidence of real-world execution, but neither source provides the ground-truth comparison set needed for rigorous precision and recall computation across a statistically adequate sample of fraud scenarios. The design science tradition supports presentation of a proof-of-concept demonstration at this stage [Geerts 2011; Sedbrook and Newmark 2008], with formal experimental evaluation deferred to subsequent research. A planned university business-school lab deployment, in which students will introduce controlled anomalies into live QBO companies operated by peer "FRPs," is designed to generate the structured precision/recall dataset that the current evaluation lacks.

Second, the anomaly injection test suite currently records pipeline pass/fail status at the level of the full five-pattern run rather than per-pattern CPI ranking output. As a result, element-level precision and recall cannot be computed from the existing test artifacts. A forthcoming revision to the test harness will instrument the detection output at the per-pattern level and is required before formal performance metrics can be published.

Third, the phi array's 17–18 element dimensionality is derived from the transaction-type ontology of QuickBooks Online and Wave Financial. Organizations using other cloud accounting platforms (Xero, FreshBooks, Sage Intacct) may have different transaction-type taxonomies, which would require a re-parameterized phi array. The generalizability of the evaluation results to those platforms has not been established.

Fourth, the LLM reasoning layer (Ollama RAG) introduces output variability that has not been systematically characterized. The language model's misappropriation risk assessments are not deterministic: the same anomaly inputs may produce qualitatively different risk narratives across model versions or inference runs. Evaluation of LLM output consistency, calibration, and alignment with ACFE taxonomy is a necessary component of a full formal evaluation and is beyond the scope of the current proof-of-concept instantiation.
