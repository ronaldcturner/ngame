Cursor Canvas Workflow for NGAME

What a Canvas Is

A .canvas.tsx file is a TypeScript/React source file that Cursor compiles on the fly into a live, scrollable document pane the user can open beside the chat. Canvases are used for rich analytical artifacts — tables, stats, structured guides, architecture reviews — where plain markdown would be a wall of text.

Where Canvases Are Stored (Two Locations)

1. Cursor's managed directory (live preview source)

~/.cursor/projects/Users-ronturner-Developer-Projects-NGAME-POC/canvases/

This is a hidden folder (~/.cursor/). Cursor serves the live canvas preview from here. It is not inside the git repository and is not visible in the Cursor file explorer.

To reveal hidden folders in Finder: press Command + Shift + . (period).

2. Project folder (git-tracked archive copy)

~/Developer/Projects/NGAME-POC/canvases/

Canvas files are manually copied here so they are tracked by git and travel with the project. The live preview is unaffected — Cursor still reads from location 1. The copy in the project folder is the backup and archive.

Keeping the Archive in Sync

After creating or updating any canvas, copy the updated file from the managed directory to the project folder:

cp -r ~/.cursor/projects/Users-ronturner-Developer-Projects-NGAME-POC/canvases \
      ~/Developer/Projects/NGAME-POC/canvases

Then commit:

git add canvases/
git commit -m "Update canvas archive"

Exporting a Canvas

Cursor has no built-in canvas export. Options:

  • Print to PDF — open the canvas pane, right-click, Print (or use the OS print dialog).
  • Markdown export — ask the agent to rewrite the canvas content as a .md file.
  • HTML export — ask the agent to produce a self-contained .html file (matches precedent of NGAME_Launcher.html and Jaccard_Presentation.html in the project root).
  • Pandoc — convert a Markdown export with pandoc guide.md -o guide.pdf or guide.docx.

Rules for the Agent

  • Always read the canvas skill at ~/.cursor/skills-cursor/canvas/SKILL.md before creating or editing any .canvas.tsx file.
  • After writing a new canvas to the managed directory, remind the user to copy it to the project folder, or offer to do the copy automatically.
  • Never create canvas files inside the project folder directly — they must be written to the managed canvases/ path first or they will not render.
